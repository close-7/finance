## 技术选型注意点
+ 构建工具（将浏览器不支持的转化为浏览器支持的）  
    1. 构建工具：gulp grunt webpack fis prepack
    2. 为什么要构建工具：资源压缩 静态资源替换 模块化处理 编译处理
+ MVVM框架选择
+ 模块化设计
    1. css模块化设计（继承，重用）
    2. js模块化设计
+ 自适应方案设计
    1. js+css
+ 代码维护及复用性设计的思考
    1. 需求变更后，代码的处理（解耦）
    2. 产品迭代 
    3. Bug定位
    4. 新功能开发
